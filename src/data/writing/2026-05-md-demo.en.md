## What's new?

Previously, I'd write all of my blog posts and project pages using JSON files. In the beginning, it made it easier to keep both the norwegian and english translation of the post/project in one file. However, this solution had some weaknesses, such as having to manually type newlines in ASCII art written in code blocks.

All blog posts and project pages need a frontmatter block at the top [(read more @ docs.page)](https://use.docs.page/frontmatter) for titles, post hooks, 'file slugs', tags and reading time.

## Block types

### Code

```ts
interface Post {
  slug: string;
  body?: ContentBlock[];
}
```

### Table

| Block     | Source       | Component       |
| --------- | ------------ | --------------- |
| `heading` | `## text`    | `<h2>`–`<h4>`   |
| `list`    | `- item`     | `<ul>` / `<ol>` |
| `callout` | `:::callout` | CardDouble      |
| `details` | `:::details` | Accordion       |

---

### Callout

:::callout[Tips]
I have also added the "callout" component from the SRCL library, which can be used by writing text inside an `:::callout[Title]` block.
:::

### Accordion

:::details[Show more]
This is only shown when the user opens the section. I add one of these by writing text in an `:::details[Title]` block.
:::
